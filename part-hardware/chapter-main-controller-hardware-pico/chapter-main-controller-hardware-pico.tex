\unnumberedChapter{A Main Controller Board}

The Raspberry PI Pico is a powerful controller with a dual core, generous program and memory sizes and USB also included. The Pico is a small board that can be soldered to a mother PCB or connected just like a big IC. This chapter will present our main controller board which is just the controller along to a CAN bus interface, the non-volatile memory and the level shifters to accommodate our 5V interface level standard. Although there will be more different controller boards in the chapters to come, this basic building block is the generic heart and can be used for many other projects in the LCS system.

Power-fail detection and brown-out detection serve complementary roles.
The power-fail detector triggers early, while the supply voltage is still within specification, allowing firmware to run at full speed and perform an orderly shutdown or state backup using stored energy from a hold-up capacitor. Brown-out detection acts as a safety backstop: if the voltage continues to fall, it forces the CPU into reset to prevent undefined execution and to protect non-volatile memory, ensuring that EEPROM or flash accesses are not performed below their safe operating voltage. Together, they enable controlled shutdown while guaranteeing fail-safe behavior during power loss.

When most of the IO pins are used in a monolithic board design, these level shifters are perhaps not needed. Since our extension connectors standardizes on 5V for digital levels, we need them here.
